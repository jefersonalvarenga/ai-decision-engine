## Data Storage

**Databases:**
- PostgreSQL (Supabase)
  - Connection: `SUPABASE_URL` and `SUPABASE_KEY` env vars
  - Region: `sa-east-1` (Brazil)
  - Project ID: `sqbidkenikaqlvdlizot`
  - Client: `supabase==2.10.0` Python SDK + `psycopg[binary]==3.2.13`
  - Schema: `public` (default)

**Tables Used by FastAPI:**
- `gk_logs` - Request log entries (INSERT via Supabase REST API)
  - Columns: remote_jid, clinic_name, detected_persona_in/out, node_executed, conversation_stage, response_message, reasoning, processing_time_ms, is_homolog, approach_used, etc.
  - Insert method: Fire-and-forget via httpx (async, non-blocking)
  - Used by: `main.py` endpoint `/v1/sdr/gatekeeper` → `_log_gk()` function

- `gk_conversations` - Active conversation state (read-only from FastAPI)
  - Used by: n8n workflow (FastAPI is stateless, doesn't write here)

- `gk_messages` - Message history (read-only from FastAPI)
  - Used by: n8n workflow (FastAPI is stateless, doesn't write here)

- `gk_leads` - Prospect queue (read-only from FastAPI)
  - Columns: approaches_tried (jsonb), final_stage, status, google_rating, google_reviews, google_ads_count
  - Used by: n8n workflow

- `gk_events` - Conversation events (read-only from FastAPI)
  - Used by: n8n workflow

- `sdr_debounce` - Message deduplication (read-only from FastAPI)
  - Used by: n8n workflow

- `sdr_config` - Global configuration (read-only from FastAPI)
  - Columns: sdr_name, environment, homolog_phone, homologJid
  - Used by: n8n workflow

**File Storage:**

## Webhooks & Callbacks

**Incoming Webhooks (from n8n):**
- `/v1/sdr/gatekeeper` - POST - Gatekeeper agent request
  - Payload: `GatekeeperRequest` (clinic_name, conversation_history, latest_message, etc.)
  - Response: `GatekeeperResponse` (response_message, should_send_message, etc.)

- `/v1/sdr/closer` - POST - Closer agent request
  - Payload: `CloserRequest` (manager_name, clinic_name, conversation_history, available_slots, etc.)
  - Response: `CloserResponse` (response_message, meeting_datetime, meeting_confirmed, etc.)

- `/v1/router` - POST - Message routing classification
  - Payload: `RouterRequest` (latest_incoming, history, intake_status, etc.)
  - Response: `RouterResponse` (intentions, reasoning, confidence, etc.)

- `/v1/reengage` - POST - Lead reengagement copywriting
  - Payload: `ReengageRequest` (lead_name, ad_source, conversation_history, etc.)
  - Response: `ReengageResponse` (generated_copy, selected_strategy, etc.)

- `/v1/utils/extract-short-name` - POST - Clinic name extraction
  - Payload: `ExtractShortNameRequest` (full_name)
  - Response: `ExtractShortNameResponse` (short_name, original_name)

**Outgoing Webhooks (from FastAPI):**
- Supabase REST API: `POST {SUPABASE_URL}/rest/v1/gk_logs`
  - Headers: apikey, Authorization (Bearer token), Content-Type: application/json
  - Used by: `_log_gk()` async function in `main.py`
  - Non-blocking: Fire-and-forget with 5s timeout

- Receptionist Simulator (test mode): `POST {webhook_url}/respond`
  - Used by: `test_full_conversation.py` when `--webhook` flag provided
  - Simulates reception responses for conversation testing
